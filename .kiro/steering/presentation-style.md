# Presentation Style Guide

## Design Philosophy
- **Visual-first**: Every concept should have a corresponding diagram or visual
- **Progressive complexity**: Build from simple to complex, never skip steps
- **Story-driven**: Frame each section as answering a question the audience has
- **Memorable**: Use analogies, contrasts, and "aha moment" structures

## Slide Principles
1. One idea per slide
2. Maximum 6 bullet points, each under 10 words
3. Every technical concept gets a "why it matters" explanation
4. Use before/after comparisons to show improvements
5. Include "try this yourself" prompts where applicable

## Content Structure Per Topic
For each major concept:
1. **The Problem** — What limitation existed before this?
2. **The Insight** — What key idea made this possible?
3. **The Paper** — Who published it, when, and where?
4. **How It Works** — Step-by-step with visuals
5. **Real Results** — Benchmarks, comparisons, production usage
6. **Limitations** — What doesn't work, edge cases, failure modes
7. **What's Next** — How this leads to the next evolution

## Tone
- Confident but honest about unknowns
- Technical but accessible (explain jargon on first use)
- Critical but constructive (don't just dismiss — explain *why*)
- Inspiring — end with "what you can build with this knowledge"

## Image/Visual Guidelines
- Reference existing diagrams from papers with proper citation
- Note which visuals to recreate vs. reference
- Include suggested image search terms for each concept
- Mark all external images with [SOURCE: url] attribution
