What Is Language Modeling?

The task of assigning a probability to a sequence of words.

P(w₁, w₂, ..., wₙ) = ∏ P(wᵢ | w₁, ..., wᵢ₋₁)

Given everything before, predict what comes next. That's it. Every LLM does this.

Step 1: Embeddings — Words Become Vectors

Each token is mapped to a point in a high-dimensional vector space (4,096–12,288 dimensions). We can only visualize 2–3, but the principle holds: geometry encodes meaning.

Mikolov et al., 2013, "Efficient Estimation of Word Representations in Vector Space," arXiv:1301.3781 · Pennington et al., 2014, "GloVe" · Visual inspired by 3Blue1Brown (2024)

Directions in Embedding Space Encode Relationships

Not random — consistent vector offsets capture semantic and syntactic relationships:

Key insight: These directions are not programmed — they emerge from training on text where these words appear in similar contexts. The model discovers the structure of language through statistics alone.

Mikolov et al., 2013 · Pennington et al., 2014, "GloVe: Global Vectors for Word Representation" · Modern LLMs use contextual embeddings (vector changes with context)

Static vs. Contextual Embeddings

Word2Vec gives ONE vector per word. But words change meaning in context.

Problem: "bank"

"I sat by the river bank" → terrain

"I went to the bank to deposit money" → financial

Same word, same static vector — wrong.

Solution: Contextual embeddings

The vector for "bank" changes depending on surrounding words.

How? Attention — let each word look at all other words to refine its representation.

This is why Transformers replaced everything: The attention mechanism gives every token a contextualized embedding that adapts to its surroundings. ELMo (2018) did this with LSTMs; Transformers (2017) did it better, faster, and in parallel.

Step 2: Attention — The Q, K, V Mechanism

For each token, compute: "Which other tokens are relevant to me? How much should I listen to each?"

Attention(Q, K, V) = softmax(QKT / √dk) · V

Vaswani et al., 2017, arXiv:1706.03762 · Intuition credit: 3Blue1Brown, "Attention in transformers, step-by-step" (2024)

Multi-Head Attention — Attending to Different Things

One attention operation captures ONE type of relationship. Real language needs many simultaneously.

Why multiple heads?

In "The cat sat on the mat because it was tired":

  • Head A might learn: "it" → "cat" (coreference)
  • Head B might learn: "tired" → "sat" (semantic cause)
  • Head C might learn: "was" → "it" (syntax: subject-verb)

These roles are not programmed — they emerge from training on billions of tokens.

The mechanics

  1. Split the embedding into h heads (each sees d/h dimensions)
  2. Attend independently per head (parallel, cheap)
  3. Concatenate all head outputs
  4. Project back to full dimension

Step 3: MLP Layers — The Knowledge Store

Attention routes information. MLPs store and retrieve knowledge.

What MLPs do

MLP(x) = W₂ · σ(W₁x + b₁) + b₂
  • W₁ (up-projection): pattern detector ("key")
  • σ (activation): non-linearity (ReLU → GeLU → SwiGLU)
  • W₂ (down-projection): information to output ("value")

~66% of GPT-3's 175B parameters are in MLP layers.

Research evidence

  • Knowledge Neurons: Specific neurons encode specific facts ("Eiffel Tower → Paris")
  • Key-Value Memory: Each MLP row acts as key (pattern) + value (output info)
  • Editing facts: Modify MLP weights → model's factual outputs change
Intuition:
Attention = telephone switchboard (who connects to whom)
MLP = the actual conversations (content being exchanged)

Geva et al., 2021, "Transformer FFN Layers Are Key-Value Memories," EMNLP, arXiv:2012.14913 · Dai et al., 2022, "Knowledge Neurons," ACL, arXiv:2104.08696 · Meng et al., 2022, "ROME"

The Complete Transformer Block

One block = Attention + MLP + Residual connections + Layer Normalization. Stack 32–128 of these.

Vaswani et al., 2017 · All major LLMs (GPT-4, Claude, Gemini, Llama, Mistral) use this exact block structure with minor variations

Known Limitations

  • O(n²) memory — 128K tokens = 16.4 billion attention entries per head per layer
  • Autoregressive — generates left-to-right, no native backtracking or planning
  • No world model — knowledge is implicit in weights, can't be verified
  • Hallucination — confident about wrong facts (no self-checking mechanism)

These limitations drive active research: Mamba (O(n)), JEPA (world models), test-time compute (planning).

Vaswani et al., 2017 · Gu & Dao, 2023 (Mamba, arXiv:2312.00752) · LeCun, 2022 (JEPA) · OpenAI o1 (2024, test-time compute)

Beyond Token Prediction: JEPA

Joint Embedding Predictive Architecture (LeCun, 2022)

Autoregressive LLMs

Predict the next token in vocabulary space

"The cat sat on the ___" → P(mat)=0.3...

  • Must predict exact surface form
  • Wastes capacity on irrelevant details
  • No explicit planning/reasoning

JEPA (LeCun's proposal)

Predict representations in abstract latent space

[abstract features] → [abstract target]

  • Discards irrelevant surface details
  • Learns world models, not word sequences
  • Basis for planning and reasoning
Status (2024-2026): I-JEPA (images, Meta 2023) and V-JEPA (video, Meta 2024) show the concept works. Not yet proven at LLM-scale for language. This is a research frontier, not a deployed solution.

LeCun, 2022, "A Path Towards Autonomous Machine Intelligence" · Assran et al., 2023, arXiv:2301.08243 (I-JEPA) · Meta AI, V-JEPA, 2024
