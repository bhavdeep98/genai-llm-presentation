LLM Architecture

What Is an LLM, Really?

A next-token prediction machine

🔍 Mystery Novel Analogy:
The story so far = context window
The clues scattered throughout = relevant tokens
"The murderer is ___" = the next token to predict

Many suspects (tokens) have probability — but only one is correct.

The architecture's job: figure out which clues matter most

Every LLM capability — reasoning, coding, translation — emerges from this single objective

Embeddings — Words as Vectors in Space

  • Words become points in high-dimensional space
  • Directions encode meaning: King − Man + Woman ≈ Queen
  • Similar words cluster together; dissimilar words are far apart
  • Modern LLMs: ~4,096–12,288 dimensions per token
Key insight: The embedding is the model's first representation. Everything after (attention, MLP) refines this initial vector.

Mikolov et al., 2013, "Efficient Estimation of Word Representations in Vector Space," arXiv:1301.3781

Attention — Who Should I Listen To?

Q, K, V Intuition (credit: 3Blue1Brown)

  • Query: "What am I looking for?"
  • Key: "What do I offer?"
  • Value: "What info do I give if you pick me?"
Attention(Q, K, V) = softmax(QKT / √dk) · V
🕵️ Detective analogy: Q = the detective's question, K = which clues respond, V = what those clues reveal

Vaswani et al., 2017, "Attention Is All You Need," arXiv:1706.03762 · 3Blue1Brown, "Attention in transformers, step-by-step," 2024

Multi-Head Attention — Multiple Perspectives

Different heads learn different relationships:

  • Head A: Syntax — "what grammatically connects"
  • Head B: Coreference — "who does 'they' refer to"
  • Head C: Semantic similarity — "related meaning"

Like 8–128 detectives each following a different lead simultaneously

MLP Layers — Where Facts Live

Attention

Routes information
"Which tokens should talk to which?"

MLP (Feed-Forward)

Stores knowledge
"Eiffel Tower → Paris"

Research finding: Factual associations are encoded in specific MLP neurons. Editing those neurons changes the model's factual outputs.

Dai et al., 2022, "Knowledge Neurons in Pretrained Transformers," ACL, arXiv:2104.08696
Geva et al., 2021, "Transformer Feed-Forward Layers Are Key-Value Memories," EMNLP, arXiv:2012.14913

The Full Transformer Architecture

Token IDs
  → Embedding + Position
    → [Attention → MLP] × N layers
      → LayerNorm → Linear → Softmax
        → Next Token Probabilities
  • One block: LayerNorm → Attention → + residual → LayerNorm → MLP → + residual
  • Repeat: 32–128 times = a full LLM
  • Decoder-only (GPT-style) is what all modern LLMs use

Why Transformers Won

(But They're Not the End)

✅ Wins

  • Fully parallelizable (100× faster than RNNs)
  • O(1) path between any two tokens
  • BLEU 28.4 EN→DE (new SOTA, 2017)

⚠️ Limitations

  • O(n²) attention — limits context length
  • Autoregressive ≠ planning
  • No explicit world model

These limitations motivate the next frontier...

Vaswani et al., 2017 · Limitations discussed in Gu & Dao, 2023 (Mamba) and LeCun, 2022 (JEPA)

Foundation Recap

  1. LLMs predict the next token from context
  2. Embeddings map words to high-dimensional vectors
  3. Attention (Q/K/V) decides which tokens matter
  4. Multi-Head captures multiple relationship types
  5. MLPs store factual knowledge
  6. Stack it all → a Transformer
  7. Transformers won but have O(n²) and planning limits
  8. JEPA proposes predicting in representation space
